# Feature Specification: 宝宝成长记录语音版

**Feature Branch**: `001-baby-tracker-voice`\
**Created**: 2026-03-22\
**Status**: Draft\
**Input**: User description: "开发一个给宝妈用的H5页面，记录宝宝日常（喂奶、换尿布、大便、辅食、睡眠），支持语音输入和AI识别，用户注册登录，数据可视化统计，治愈系可爱风格"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 语音记录宝宝日常 (Priority: P1)

宝妈通过长按语音按钮，用自然语言描述宝宝的日常活动，系统自动识别并生成结构化记录。例如宝妈说"宝宝刚才喝了150毫升奶"，系统识别为喂奶记录，时间默认为当前时间。

**Why this priority**: 这是应用的核心价值主张，语音输入大幅降低记录门槛，让忙碌的宝妈能够快速记录，是整个产品的基础功能。

**Independent Test**: 可以独立测试语音输入到记录生成的完整流程，验证AI识别准确性和记录正确性，无需依赖其他功能即可验证核心价值。

**Acceptance Scenarios**:

1. **Given** 宝妈已登录并进入主页面，**When** 长按语音按钮并说"宝宝刚才喝了150毫升奶"，**Then** 系统识别为喂奶记录，显示奶瓶emoji，记录时间默认为当前时间，奶量150ml
2. **Given** 宝妈已登录并进入主页面，**When** 长按语音按钮并说"昨天中午宝宝拉粑粑了，是黄色的软便"，**Then** 系统识别为大便记录，显示便便emoji，日期为昨天，时间为中午，状态为"黄色软便"
3. **Given** 宝妈已登录并进入主页面，**When** 长按语音按钮并说"宝宝刚才换尿布了"，**Then** 系统识别为换尿布记录，显示尿布emoji，时间为当前时间
4. **Given** 宝妈已登录并进入主页面，**When** 长按语音按钮并说"宝宝下午2点睡了3个小时"，**Then** 系统识别为睡眠记录，显示睡觉emoji，开始时间为下午2点，持续时长3小时
5. **Given** 宝妈语音描述缺少关键信息（如只说"宝宝喝奶了"未说明奶量），**When** 系统检测到信息不完整，**Then** 悬浮对话框主动询问"请问宝宝喝了多少毫升呢？"，宝妈语音回答后完成记录
6. **Given** 系统需要补充信息，**When** 显示悬浮对话框询问，**Then** 对话限制在两轮内完成，提供确认按钮供宝妈确认记录内容

***

### User Story 2 - 记录查看与筛选 (Priority: P1)

宝妈可以查看所有历史记录，按时间排序，并支持按事项类型筛选和按时间范围筛选。

**Why this priority**: 记录的价值在于回顾和分析，查看功能是语音记录的配套核心功能，让用户能够追踪宝宝的成长轨迹。

**Independent Test**: 可以独立测试记录的展示、排序、筛选功能，验证数据组织和展示逻辑，不依赖语音输入功能（可使用测试数据）。

**Acceptance Scenarios**:

1. **Given** 宝妈已登录并有历史记录，**When** 进入主页面，**Then** 所有记录按时间倒序排列显示，最新记录在最上方
2. **Given** 宝妈在主页面查看记录列表，**When** 点击筛选按钮选择"只看喂奶"，**Then** 列表仅显示喂奶类型的记录
3. **Given** 宝妈在主页面查看记录列表，**When** 选择时间筛选"最近7天"，**Then** 列表仅显示最近7天内的记录
4. **Given** 宝妈在主页面查看记录列表，**When** 同时选择"只看睡眠"和"最近3天"，**Then** 列表显示最近3天内的睡眠记录

***

### User Story 3 - 数据统计图表 (Priority: P2)

宝妈可以通过左上角的按钮进入统计页面，查看近一个月的宝宝数据可视化图表，帮助了解宝宝的规律和趋势。

**Why this priority**: 数据统计提供了更高层次的价值，帮助宝妈发现宝宝的作息规律，但属于增值功能，不是核心记录功能的必需品。

**Independent Test**: 可以独立测试图表展示功能，验证数据聚合和可视化正确性，使用测试数据即可验证。

**Acceptance Scenarios**:

1. **Given** 宝妈已登录并有近一个月的数据，**When** 点击左上角统计按钮，**Then** 进入统计页面显示近一个月奶量趋势折线图
2. **Given** 宝妈在统计页面，**When** 查看图表，**Then** 可以看到每天换尿布次数折线图
3. **Given** 宝妈在统计页面，**When** 查看图表，**Then** 可以看到每天大便次数折线图
4. **Given** 宝妈在统计页面，**When** 查看图表，**Then** 可以看到每天睡眠总时长趋势图
5. **Given** 宝妈在统计页面，**When** 查看图表，**Then** 可以看到辅食摄入种类分布图

***

### User Story 4 - 用户注册与登录 (Priority: P1)

宝妈可以通过注册创建账户，并使用账户登录系统，个人数据与账户绑定。

**Why this priority**: 用户账户是数据持久化和多设备同步的基础，虽然可以先用本地存储做MVP，但注册登录是产品完整性的必要功能。

**Independent Test**: 可以独立测试注册登录流程，验证账户创建、密码加密、登录认证功能，不依赖其他业务功能。

**Acceptance Scenarios**:

1. **Given** 宝妈未注册，**When** 输入手机号/邮箱和密码点击注册，**Then** 账户创建成功，密码加密存储，自动登录进入主页面
2. **Given** 宝妈已注册，**When** 输入正确的账号密码点击登录，**Then** 登录成功进入主页面，显示个人记录数据
3. **Given** 宝妈已登录，**When** 点击退出登录，**Then** 退出到登录页面，再次进入需要重新登录
4. **Given** 宝妈登录时输入错误密码，**When** 点击登录，**Then** 提示"账号或密码错误"，不泄露具体是账号还是密码错误

***

### Edge Cases

- **网络不稳定时语音输入如何处理？** → 系统返回"网络异常"提示，建议用户检查网络后重试
- **同一天内多次相同类型记录（如多次喂奶）如何区分展示？** → 通过发生时间区分，时间线是主页重要展示项目，所有记录按照事件发生时间倒序排序（最新的在最上方）
- **语音识别的置信度较低时如何处理？** → 系统再次询问用户确认，或记录到"其他"类型供用户后续手动分类
- **用户长时间未操作会话过期如何处理？** → 会话过期后自动退出到登录页面，用户需重新登录
- **宝宝数据跨天如何计算（如睡眠从晚上11点到早上6点）？** → 跨天睡眠时间算作第二天的睡眠时间（以睡眠结束时间为准）
- **删除记录后的数据一致性如何保证？** → 删除操作同步更新相关统计数据，确保数据一致性

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 支持用户通过手机号或邮箱注册账户
- **FR-002**: 系统 MUST 使用加密算法（如bcrypt）存储用户密码，禁止明文存储
- **FR-003**: 系统 MUST 支持用户通过账号密码登录
- **FR-004**: 系统 MUST 提供长按语音输入按钮，支持语音采集
- **FR-005**: 系统 MUST 使用AI大模型识别语音内容，提取记录类型、时间、数量、状态等关键信息
- **FR-006**: 系统 MUST 支持识别的记录类型包括：喂奶（奶量ml）、换尿布、大便（状态描述）、辅食（种类、数量）、睡眠（开始时间、持续时长）、其他（用于无法识别的内容）
- **FR-007**: 系统 MUST 智能解析时间表达，支持"刚才"、"昨天"、"上午"、"下午2点"等模糊时间，默认使用当前时间
- **FR-008**: 当AI检测到关键信息缺失时，系统 MUST 通过悬浮对话框主动询问用户，对话限制在两轮内
- **FR-009**: 系统 MUST 在主页面按时间倒序展示所有记录
- **FR-010**: 系统 MUST 支持按记录类型筛选（喂奶、换尿布、大便、辅食、睡眠）
- **FR-011**: 系统 MUST 支持按时间范围筛选（今天、最近3天、最近7天、最近30天）
- **FR-012**: 系统 MUST 使用emoji图标区分不同记录类型（如🍼喂奶、💩大便、😴睡眠等）
- **FR-013**: 系统 MUST 提供统计页面入口（左上角按钮）
- **FR-014**: 统计页面 MUST 展示近一个月的数据可视化图表
- **FR-015**: 统计图表 MUST 包括：奶量趋势折线图、换尿布次数折线图、大便次数折线图、睡眠时长趋势图、辅食种类分布图
- **FR-016**: 系统 MUST 采用可爱、治愈系、暖萌的视觉风格
- **FR-017**: 当网络不稳定导致语音输入失败时，系统 MUST 显示"网络异常"提示，建议用户检查网络后重试
- **FR-018**: 当AI语音识别置信度低于阈值时，系统 MUST 再次询问用户确认或将记录归类到"其他"类型
- **FR-019**: 跨天睡眠时间（如23:00-06:00）MUST 算作第二天的睡眠时间（以睡眠结束时间为准）

### Key Entities

- **User（用户）**: 代表宝妈用户，包含账号信息（手机号/邮箱）、加密密码、创建时间
- **BabyRecord（宝宝记录）**: 代表一条宝宝日常记录，包含记录类型（喂奶/换尿布/大便/辅食/睡眠）、记录时间、详细信息（根据类型不同而变化，如奶量、状态描述、时长等）、关联的用户ID、创建时间
- **RecordType（记录类型枚举）**: FEEDING（喂奶）、DIAPER（换尿布）、POOP（大便）、FOOD（辅食）、SLEEP（睡眠）、OTHER（其他）

## Assumptions

- 用户主要使用中文进行语音输入
- 目标用户为0-3岁宝宝的宝妈
- 设备支持H5页面和麦克风权限
- 网络环境以WiFi和4G/5G为主
- 数据存储采用云端方案，支持多设备同步
- AI语音识别服务可用且有合理的API调用成本

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户能够在3秒内完成语音输入并看到AI识别结果
- **SC-002**: AI语音识别准确率不低于85%（针对宝宝日常场景关键词）
- **SC-003**: 从语音输入到记录生成的完整流程平均耗时不超过10秒（包括必要的补充对话）
- **SC-004**: 用户能够在2次点击内找到任意历史记录
- **SC-005**: 统计页面图表加载时间不超过3秒
- **SC-006**: 90%的用户能够独立完成首次语音记录而无需查看帮助文档
- **SC-007**: 系统支持至少1000条记录的数据量而不影响查询性能
